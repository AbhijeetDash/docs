# Override CLI Commands Documentation

This document provides an overview of the available CLI commands for the \`oi\` application, detailing their responsibilities, options, and usage.

## Initialize Command

The \`init\` command is responsible for setting up the initial configuration for a new \`oi\` project. It creates the project configuration file (\`oi-config.json\`), manages the ignore list, and provides instructions for further setup.

<div data-node-id="ef5a8a96-c2b8-4adc-845f-60a6d83b93eb" activeindex="0" tabs="Request 1" data-type="custom-request" class="custom-request">
<div tabindex="0" data-type="request-content" class="request-content"><pre data-node-id="519566c9-e762-49d0-a8f8-fa0478d6d288"><code class="language-oi">init</code></pre></div>
</div>

\*\*Options:\*\*

- \-o, --output &lt;path&gt;: Specify a custom configuration file path
- \-i, --ignore &lt;files...&gt;: Specify files or directories to ignore
- \-v, --verbose: Enable verbose output
- \-n, --project-name &lt;name&gt;: Specify a project name

\*\*Responsibilities:\*\*

- Display ASCII art to indicate successful initialization
- Add files to the ignore list in the configuration
- Create and initialize the project configuration
- Perform a dry run of the initialization process without making changes

## Start Command

The \`start\` command initiates the file-watching mechanism that monitors specific files for changes related to prompts in the \`oi\` project.

<div data-node-id="ff020140-e67b-4196-9f24-2d6c5751c4f5" activeindex="0" tabs="Request 1" data-type="custom-request" class="custom-request">
<div tabindex="0" data-type="request-content" class="request-content"><pre data-node-id="22066e8c-d147-400d-a2a1-e44655c8b18a"><code class="language-oi">start</code></pre></div>
</div>

\*\*Options:\*\*

- \-v, --verbose: Enable verbose output

\*\*Responsibilities:\*\*

- Start watching files for prompt changes
- Handle verbose mode for detailed logging
- Catch and handle any errors during the watch process

## Config Command

The \`config\` command updates project settings in \`oi-config.json\`, manages global configurations, and allows users to select an active platform.

<div data-node-id="c7ce4451-51b7-4f87-aec4-ce185547a209" activeindex="0" tabs="Request 1" data-type="custom-request" class="custom-request">
<div tabindex="0" data-type="request-content" class="request-content"><pre data-node-id="fca647ca-d2ad-4922-87f7-5aa3db991c70"><code class="language-oi">config</code></pre></div>
</div>

**Options:**

- \-n, --project-name &lt;name&gt;: Update project name
- \-i, --ignore &lt;files...&gt;: Specify files or directories to ignore
- \-g, --global: Set global variables like API keys and org IDs
- \-sa, --set-active: Set active platform

**Responsibilities:**

- Prompt the user for platform-specific configuration details
- Manage global configuration, including setting the active platform
- Handle local configuration updates, including project name and ignored files

Use the `--verbose` option with any command to get detailed logging information during execution.
</div>
